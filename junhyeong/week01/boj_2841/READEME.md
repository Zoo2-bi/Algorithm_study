## 🚀 접근 방법

### 어떤 방식으로 접근했는지

#### 행을 배열로 변환

- 첫번째 요소는 줄
- 두번째 요소는 플랫

#### 플랫을 스택에 저장

- 플랫을 아래의 조건에 따라 스택에 저장
  1. 스택의 top보다 큰 경우 push
  2. 스택의 top보다 작은 경우 push하지 않고 pop
     - 이 때, 해당 순서의 플랫이 top보다 클 때 까지 반복
  3. 스택의 top과 같은 경우 continue

#### 줄이 달라지는 경우

- 줄은 6줄이므로 6개의 스택을 생성
- 이 때, 길이가 7인 2차원 배열을 생성
- 해당 배열의 인덱스가 줄의 번호

### 핵심 로직

#### 첫번째 순서에는 움직임 발생

- 조건문으로 첫번재 항이라면 반드시 움직임 추가
- 스택에 플랫 추가

#### 반복문 수행 내용

- 두번째 움직임 부터는 첫번째의 줄과 비교하여 줄이 같다면 플랫을 확인
- 플랫이 이전 플랫보다 클 경우 움직임 발생
- 플랫이 이전 플랫보다 작을 경우, stack의 top과 비교하여, top이 작을때까지 pop
  - pop 횟수만큼 움직임 발생
  - 해당 순서의 플랫을 잡으면서 움직임 발생
  - 즉, pop횟수 + 1 만큼 움직임 발생

## ⏱️ 시간 복잡도

- 시간: O( n )
  - for문 내부에 while문이 있어서 n^으로 보이지만,
  - while문의 반복 횟수는 N 이하임 (즉, while문의 반복회수 <= N)
  - n^인 경우는, 매번 N번을 도는 경우
- 공간: O( n )
  - 스택의 최대 길이는 N이며, 6N이므로 n

## 🌅 느낀 점

### 배운 점

- 다차원 배열이 필요한 상황을 인지
  - 스택이 여러개 필요한데, 해당 스택의 개수가 정해져 있음
  - 배열의 요소를 배열로 하는 2차원배열을 이용하면 되겠구나!
- 조건 분기
  1. plat이 stack의 top 보다 큰 경우
  2. plat이 stack의 top과 같은 경우
  3. plat이 stack의 top 보다 작은 경우

### 아쉬운 점

- 2차원 배열을 ES6문법으로 생성하는 방법이 생소하게 느껴짐
- line, plat을 숫자형으로 가공하는 것을 깜빡해서 오답이 여러번 뜸
- while문의 조건문 부분에 대한 생각을 짧게한 나머지 오답이 여러번 뜸
  - <를 충족하지 않 을때, ===와 > 두가지로 또 나눠서 생각해봐야함

### 개선 포인트
