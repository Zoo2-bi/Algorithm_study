## 🧩 문제 정보

- 플랫폼: BOJ
- 문제 번호: 10828
- 문제 이름: 스택
- 링크: https://www.acmicpc.net/problem/10828

## 💡 문제 설명

- 해당 프로그래밍 언어의 문법으로 스택이라는 자료구조의 기본 연산을 구현하고 주어진 명령을 순차적으로 처리하는 문제

## 🚀 접근 방법

- 입력 예제의 첫번재 줄을 정수 n으로 받고
- 나머지 항들을 n을 통해 반복문으로 받으며 switch문으로 해당 명령에 맞는 행위를 수행
- 반복문 내부에서 조건물을 사용할 경우 가독성이 좋지 않을 것이라 판단하여 switch문을 사용

## ⏱️ 시간 복잡도

- 시간: O( N )
  - 반복문이 1번 존재하므로
- 공간: O( N )
  - lines와 arr가 각각 O(N)이므로

## 🌅 느낀 점

### 배운 점

#### 느낀점

- 알고리즘을 푸는 언어의 문법 및 자료구조를 다루는 기본기를 다잡을 수 있는 문제였다.
- 스택의 특징인 FILO(First In Last Out) 특징을 인지하며 문제를 푸는 과정은 스택이라는 자료구조를 이해하는 기초적인 과정이었다.

#### 문법

1. push() 메서드: 배열의 뒤쪽에 요소를 추가하는 메서드
2. pop() 메서드: 배열의 뒤쪽의 요소를 추출하여 반환하는 메서드
3. unshift() 메서드
   - 배열의 앞쪽에 요소를 추가하는 메서드
   - 배열 전체의 요소들을 한칸씩 뒤로 미는 과정 발생 (시간 복잡도 N)
4. shift() 메서드
   - 배열의 앞쪽에 요소를 삭제하는 메서드
   - 배열 전체의 요소들을 한칸씩 당기는 과정 발생 (시간 복잡도 N)
5. split() 메서드
   - 문자열을 배열로 반환 (새로운 배열을 반환)
   - 원본을 변경하지 않음 (문자열은 ummutable이므로)

#### 캐리지 리턴

- OS별로 줄바꿈 방식의 차이가 있음
  1. Windows: /r/n
  2. Lunux: /n
- 이 때, 백준은 Linux 기반이지만, 본인은 Windows를 사용중이므로 입력 예제가 여러줄일 때 각자 다른 값이 넘어옴
- lines에 값을 할당할 때 사용한 trim()메서드는 맨앞/뒤의 공백만 제거해줌 -> 각 요소를 /n으로 잘랐기 때문에 Windows에서의 lines에는 맨 처음/끝 요소를 제외한 요소에 /r이 남아있음
- 따라서 lines 배열에 요소를 할당할 때 trim()을 한번더 적용해주거나, lines의 요소를 사용할 때 trim()을 적용시켜줘야함

#### 출력 방식

- 자바스크립트에서 console.log()는 반복되면 성능이 느려짐
- 따라서 문자열을 모았다가 join()메서드를 통해 한 번 출력하는 패턴이 사실상 필수적

### 개선 포인트

- shift(), unshift(), pop(), push()의 의미와 사용 상황을 확실하게 인지하고 사용할 수 있게 되기.
- console.log() 최소화 -> 마지막에 join메서드와 함께 한번만 호출
- lines 할당 시 split(/n)에서 split(/r?/n)으로 수정
  - ?이란 앞의 값이 있어도, 없어도 된다는 의미
  - - : 0개 이상
  - - : 1개 이상
  - ? : 0개 또는 1개
