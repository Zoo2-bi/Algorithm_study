## 💡 문제 설명

- 문제에 대한 간단한 요약 (내가 이 문제를 얼마나 파악하고 있는지 확인)

- 쇠막대기 사이의 레이저 개수 + 1이 잘려진 쇠막대기의 개수
- 여러개의 쇠막대기들이 각각 몇개로 잘렸는지 확인하는 문제
- 스택 자료구조를 이용하여 풀이

---

## 🚀 접근 방법

- 어떤 방식으로 접근했는지
  - "(" 다음에 ")"가 오면 레이저로 인식
  - ")" 다음에 ")"가 오면 쇠막대기로 인식
  - 막대기의 개수를 세는 타이밍 고민
    1. 레이저를 인식하는 순간에 막대기를 세는게 맞는지
    2. 쇠막대기를 인식하는 순간에 막대기를 세는게 맞는지
    3. pop하는 순간마다 세는게 맞는지
  - pop하는 순간마다 세는 것이 맞다고 판단
    - 레이저를 인식하는 순간에 센다면, 막대기가 끝났을 때를 못 셈
    - 쇠막대기를 기준으로 센다면, 막대기가 시작될 때 마다 해당 막대기에 대한 개수를 담아야함 -> 메모리 공간 낭비

- 핵심 로직
  - pop하는 순간 마다 막대기를 셈
    1. 레이저를 인식하는 경우: "("의 개수 만큼 === stack의 길이 만큼 answer 증가
    2. 쇠막대기를 인식하는 경우: 1만큼 answer 증가
  - 즉, pop하는 순간에 집중

---

## ⏱️ 시간 복잡도

- 시간: O( n )
  - 쇠막대기의 길이(n) 만큼 반복문을 수행하므로
  - stack에서 pushm pop은 O(1)
- 공간: O( n )
  - stack은 최대 n (전부 "("일 때)
