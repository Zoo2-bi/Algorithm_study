## 💡 문제 설명

- 문제에 대한 간단한 요약 (내가 이 문제를 얼마나 파악하고 있는지 확인)

- VPS 문자열일 경우 YES, 아닌 경우 NO를 출력
- VPS란, 한쌍의 괄 기호로 된 문자열들로 구성된 괄호 문자열
- 스택을 통해 VPS 판단

---

## 🚀 접근 방법

### 어떤 방식으로 접근했는지

#### 스택을 활용

- "("인 경우 스택에 push
- ")"인 경우 스택을 확인

### 핵심 로직

- ")"인 경우 스택을 확인
  - 스택이 비어있지 않은 경우 스택에서 pop
  - 스택이 비어있는 경우 NO이므로 break

- YES인 경우
  - for문을 전부 돌고난 뒤의 stack이 비어있면 YES

- NO인 경우
  - for문을 전부 돌기 전에 ")"에서 pop 불가시 NO
  - for문을 모두 돌고 난 후 stack이 비어있지 않으면 NO

---

## ⏱️ 시간 복잡도

- 시간: O( n )
  - 테스트 케이스 T와 문자열의 길이 k의 곱이므로 n
- 공간: O( )
  - 길이가 k인 스택 1개를 사용하므로 n

## 🌅 느낀 점

### 배운 점

- 중간에 NO인 경우 answer에 바로 NO를 삽입하지 않고 break만 수행
- for을 빠져나와서, 조건을 확인하여 YES, NO 판단

### 아쉬운 점

- lines[i]에 trim()메서드를 깜빡하여 맞는 풀이임에도 계속 해맴
- lines 할당의 trim()메서드는 앞,뒤만 제거해줌
- split("\n")은 "\r"를 제거해주지 않음

### 개선 포인트

- 조건문이 복잡할 경우 삼항 연산자로 바꿔보자!
