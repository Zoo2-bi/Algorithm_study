## 💡 문제 설명
<!--문제에 대한 간단한 요약 (내가 이 문제를 얼마나 파악하고 있는지 확인)-->

- 기본 Stack에 추가 기능을 넣어 구현하는 문제

---

## 🚀 접근 방법
<!--
- 어떤 방식으로 접근했는지
- 왜 이 알고리즘을 선택했는지
- 핵심 로직
-->

- 자바의 경우 Stack 라이브러리를 이용해서 구현하면 되므로 그렇게 진행했다.
- 몇 부분에 추가적인 기능이 필요해 그 부분만 유의하면 된다.
  - pop: stack이 비었을 경우 -1 출력
  - empty: stack이 비었을 경우 1, 아닐 경우 0 출력
  - top: stack이 비었을 경우 -1 출력, 아닐 경우 peek 출력

---

## ⏱️ 시간 복잡도

- 시간: O(n)
- 공간: O(n)

## 🌅 느낀 점
<!--배운 점 / 아쉬운 점 / 개선 포인트-->

stack 라이브러리를 사용하는 것이 아니라,
실제로 배열을 통해 직접 stack을 만들어야 하는
문제라는 생각도 들지만 직접 구현하는 것이
라이브러리보다 당연 효율 면에서 떨어질 게 분명하여
항상 직접 구현을 하지 않게 된다.
그 점이 조금 아쉽긴 하다.

하지만 stack 라이브러리 사용법을 익히는 데엔 좋은 문제다.
