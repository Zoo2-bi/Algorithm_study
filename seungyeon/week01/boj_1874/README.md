- 문제 이름: 스택 수열
- 링크: https://www.acmicpc.net/problem/1874

---

## 💡 문제 설명
<!--문제에 대한 간단한 요약 (내가 이 문제를 얼마나 파악하고 있는지 확인)-->

- 1부터 n까지의 수를 stack에 넣으면서 push,pop 해서 주어진 수열을 만들 수 있는지 판단
  - push 순서는 오름차순
  - pop으로 수열에 필요한 숫자들을 빼냄
  - 가능하다면 push, pop을 어떤 순서로 해야하는지 한 줄씩 출력
  - 불가능하다면 NO 출력

---

## 🚀 접근 방법
<!--
- 어떤 방식으로 접근했는지
- 왜 이 알고리즘을 선택했는지
- 핵심 로직
-->

- 1부터 n까지 반복을 돌림
  - 아님! 연속으로 pop할 때도 있어서 for문으로 처리 불가능
    - for문에서 다음 숫자 넣기를 정지하기 까다로움
    - 4와 3이 연속으로 pop이며 push를 하면 안 되는데 for문에서 구현하려면
    <br>`4 pop하면 i--해서 i를 4로 돌려놓고 push를 안 함, 다음에 i++해서 5가 되었을 때 비교해서 3 pop하고 i--해서 i를 4로 돌려놓고 push를 안 함...`
  - while문으로 넣을 숫자와 읽을 숫자 한도를 정하고 돌리기
- 주어진 수열의 수까지 push, pop
- 다음 수를 보고 push 혹은 pop
- 불가능할 경우 NO 출력

---

## ⏱️ 시간 복잡도

- 시간: O(n) - whlie문 2n+command문 2n
- 공간: O(n) - stack n+리스트 2n

## 🌅 느낀 점
<!--배운 점 / 아쉬운 점 / 개선 포인트-->

괄호 문제와 같다고 생각했지만 의외로 구현에서
추가적으로 생각해야할 부분이 많았다.
(for문과 while 문의 선택이라든가,
비교할 수열 count 및 넣을 수 count 판단 등
-특히 내 코드에선 `count<=n`이 필요했음)

예전에 풀었을 땐 수열의 다음 수가 stack의 peek보다 작을 경우 바로 break 및 종료까지 구현해놔서
이게 더 효율적일거라 생각했지만 그렇지도 않았다. (지금 새로 푼 게 더 성능이 좋다.)
