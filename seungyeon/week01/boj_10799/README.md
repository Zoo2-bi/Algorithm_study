- 링크: https://www.acmicpc.net/problem/10799

---

## 💡 문제 설명
<!--문제에 대한 간단한 요약 (내가 이 문제를 얼마나 파악하고 있는지 확인)-->

- `()`로 쇠막대기의 길이를 알려주거나 쇠막대기를 자른다.
- 최종적으로 몇 개의 쇠막대기가 나오는지 구하는 문제.

---

## 🚀 접근 방법
<!--
- 어떤 방식으로 접근했는지
- 왜 이 알고리즘을 선택했는지
- 핵심 로직
-->

- 경우와 규칙만 잘 찾아도 풀린다.
  - 이전이 `(`인데 다음이 `(`일 경우: 막대 추가
  - 이전이 `(`인데 다음이 `)`일 경우: 막대 자르기=기존 쇠막대기 조각 수+현재 막대 수
  - 이전이 `)`인데 다음이 `)`일 경우: 막대 삭제
- 다만, 이 문제는 꼭 stack으로 풀지 않아도 된다.
  - 한 문자열을 훑으며 이전 글자와 비교하면 풀 수 있다.
  - 그렇게 한 문자열에서 해결하는 게 더 성능상 좋다.

---

## ⏱️ 시간 복잡도

- 시간: O(n)
- 공간: O(n)

## 🌅 느낀 점
<!--배운 점 / 아쉬운 점 / 개선 포인트-->

이전에 한 번 풀어봤던 문제라 규칙을 떠올리는 건 쉬웠다.
하지만 굳이 stack을 쓰지 않아도 된다는 걸 떠올리지 못해
괜히 무겁고 복잡하게 만든 것 같다.

한 문자열 내에서 chatAt으로 해결하면 공간복잡도도 O(1)이 나오고 속도도 더 빨라지고 코드도 더 간단해진다.
이런 식으로 개선이 가능함을 생각해둘 것.
