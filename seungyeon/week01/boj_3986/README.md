## 🧩 문제 정보

- 플랫폼: BOJ
- 문제 번호: 3986
- 문제 이름: 좋은 단어
- 링크: https://www.acmicpc.net/problem/3986

---

## 💡 문제 설명
<!--문제에 대한 간단한 요약 (내가 이 문제를 얼마나 파악하고 있는지 확인)-->

- A와 B의 문자열을 넣고 A끼리, B끼리 쌍을 짓는다.
- 지어진 쌍이 서로 교차하지 않으면 좋은 단어.

---

## 🚀 접근 방법
<!--
- 어떤 방식으로 접근했는지
- 왜 이 알고리즘을 선택했는지
- 핵심 로직
-->

- 괄호와 비슷하게 쌍이 이루어지는지 stack을 통해 확인
  - A가 나왔을 때 이전이 A면 pop
  - B가 나왔을 때도 마찬가지
- pop 되지 않으면 일단 push
- 최종적으로 남는 글자가 없으면 좋은 단어

---

- 시간: O(n)
- 공간: O(n)
