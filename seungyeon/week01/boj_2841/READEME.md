## 💡 문제 설명
<!--문제에 대한 간단한 요약 (내가 이 문제를 얼마나 파악하고 있는지 확인)-->

- 기타엔 여러 줄이 있고 줄마다 프렛이 n개 존재함
- 한 줄에서 여러 프렛을 누르고 있다면 가장 높은 프렛의 음이 발생
- 이렇게 프렛을 누르거나 떼는 것을 손가락 한 번 움직인 걸로 침
- 손가락을 가장 적게 움직이는 횟수 구하기

---

## 🚀 접근 방법
<!--
- 어떤 방식으로 접근했는지
- 왜 이 알고리즘을 선택했는지
- 핵심 로직
-->

- 줄마다 stack을 생성
- 줄에 숫자가 들어왔을 때 peek와 같으면 무시, 아니면 push
- 만약 들어온 숫자가 peek보다 작으면 peek부터 pop하기 시작
- push pop 횟수를 저장했다가 출력

---

## ⏱️ 시간 복잡도

- 시간: O(n) - n만큼 스택에 들어가고 나옴=2n
- 공간: O(n) - `줄 배열 l+스택 내부 n, n개가 모든 스택에 중복으로 들어가는 게 아니라 나뉘어 들어가기 때문에 +` =n+l

## 🌅 느낀 점
<!--배운 점 / 아쉬운 점 / 개선 포인트-->

실버1이라 간만에 하는 입장에서 나름 긴장했는데 그렇게 어렵진 않았던 것 같다.
문제는 이해하는데 걸린 시간이 더 길었다.

다만... stack을 줄 개수만큼 만들다보니 성능은 확실하게 안 좋다. (395496kb, 1632ms)
이 부분이 무척이나 아쉽다.

개선 포인트는 다음과 같이 짚을 수 있겠다.
- 줄 개수가 n개로 정해져 있으니 ArrayList<Stack> 말고 Stack[] 배열 사용
- Stack은 멀티스레드 환경을 고려한 동기화 처리가 되어 있어서 무겁다고 하니 ArrayDeque 사용

이렇게 하면 백준 채점 시간이 1/3 이하로 단축될거라고 한다. (테스트는 안 해봄)
