## 💡 문제 설명
<!--문제에 대한 간단한 요약 (내가 이 문제를 얼마나 파악하고 있는지 확인)-->

- 1번부터 N번까지 N명의 사람이 원으로 앉음
- K번째 사람 제거를 반복
- 제거되는 순서(요세푸스 순열)을 출력

## ⏱️ 시간 복잡도

- 시간: O(n) - k번씩 빼야 하므로 (1+k)n => kn
- 공간: O(n) - string builder도 있으니 2n => n

---

## 🌅 느낀 점
<!--배운 점 / 아쉬운 점 / 개선 포인트-->

Queue를 사용하는 기본적인 문제라 크게 배웠다고 할 건 없고 개선 가능 포인트를 적어보겠다.

n과 k의 범위가 매우 커진다면(둘 다 10만 개일 경우) nk 방식은 시간이 오래 걸릴 수 있다. 그럴 땐 다음의 방법을 쓰면 좋다고 한다.
- LinkedList 사용: List에서 직접 인덱스를 계산해 k위치의 원소를 제거하면 시간을 줄일 수 있음.
- Segment Tree: O(n log n)으로 해결 가능.
